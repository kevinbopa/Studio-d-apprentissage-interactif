Section 1.3
Les 12 Principes agiles
Opérationnalisation des 4 valeurs dans le travail quotidien d'équipe.
Définition
12 principes opérationnalisent les valeurs dans le travail quotidien des équipes agiles.
Version simplifiée
Livrer souvent, collaborer en continu et améliorer la qualité techniquement.
Explication détaillée
Les principes couvrent l'orientation client, la collaboration directe, la mesure par fonctionnalités livrées, le rythme soutenable, la simplicité, l'auto-organisation et l'amélioration continue. Ils transforment les valeurs abstraites en comportements observables au quotidien.
Exemple concret
Une équipe tient une revue de sprint, puis ajuste immédiatement le backlog avec le client selon les retours reçus.
Points clés
Le progrès est mesuré par ce qui fonctionne.
Le rythme soutenable évite l'épuisement.
L'excellence technique soutient l'agilité.
Erreurs fréquentes à l'examen
Multiplier les livraisons sans boucle de feedback réelle.
Négliger la dette technique au profit de la vitesse.
Résumé de section
Les principes transforment l'agilité en pratiques concrètes d'équipe — ils sont la colonne vertébrale opérationnelle du Manifeste.

// Calcule la vélocité moyenne d'une équipe Scrum
function calculateVelocity(sprintPoints) {
  if (sprintPoints.length === 0) return 0;
  const total = sprintPoints
    .reduce((acc, pts) => acc + pts, 0);
  return Math.round(total / sprintPoints.length);
}
